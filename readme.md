# Position工具使用文档

## 1. 软件介绍

Position工具是一个专门用于编辑和生成JSON格式位置数据的工具，主要用于游戏或动画中的位置路径编辑。

### 主要功能
- 可视化编辑多个事件的位置路径
- 支持BPM模式和毫秒模式的时间控制
- 提供多种缓动函数选择
- 支持向量操作（镜像、旋转、平移、缩放）
- 事件分组管理（支持展开/折叠）
- 实时预览路径效果
- 退出时自动保存配置
- 一键生成JSON格式输出

## 2. 安装与启动

### 安装
1. 从提供的安装包中解压文件到任意目录
2. 运行 `JSON Position Tool.exe` 启动应用

### 启动
启动应用后，会先显示一个启动动画，然后进入主界面。

**配置自动加载**：
- 如果上次退出时选择了保存配置，应用会自动加载上次的工作状态
- 包括所有事件数据、分组信息和BPM模式设置

## 3. 主界面介绍

### 界面布局
- **左侧面板**：事件列表和编辑区域
- **右侧面板**：JSON输出、播放控制和预览区域

### 顶部按钮
- **全屏**：点击后应用进入全屏模式
- **退出**：退出应用，退出前会询问是否保存配置

## 4. 基本操作

### 添加事件
1. 在左侧面板点击 "事件" 标题旁边的 "+ 添加" 按钮
2. 新事件会自动添加到列表末尾
3. 新事件的开始时间会继承前一事件的结束时间
4. 新事件的结束时间会延续前一事件的持续时长

### 编辑事件
1. 选择要编辑的事件
2. 在事件编辑区域修改以下参数：
   - **开始时间**：事件开始的时间点
   - **结束时间**：事件结束的时间点
   - **起点**：事件开始的坐标位置
   - **终点**：事件结束的坐标位置
   - **缓动**：选择位置变化的缓动函数
   - **链接**：勾选后，当前事件的开始时间和起点会自动继承上个事件的结束时间与终点

### 删除事件
1. 选择要删除的事件
2. 点击事件右侧的 "删除" 按钮

## 5. 时间模式

### 毫秒模式
- 时间以毫秒为单位显示和输入
- 适合需要精确时间控制的场景

### BPM模式
- 时间以拍为单位显示和输入
- 可以通过调整BPM值来改变实际时间
- 当BPM值变化时，拍数保持不变，实际时间会相应调整
- 预览区域会显示正确的总节拍数

## 6. 向量操作

### 单个事件操作
1. 选择要操作的事件
2. 在 "向量操作" 区域选择操作类型：
   - **水平镜像**：沿水平轴镜像位置
   - **垂直镜像**：沿垂直轴镜像位置
   - **对角镜像**：沿对角线镜像位置
   - **旋转**：输入旋转角度和旋转中心
   - **平移**：输入X和Y方向的偏移量
   - **缩放**：输入X和Y方向的缩放比例
3. 点击 "执行" 按钮应用操作
4. 执行向量操作后，当前事件和下一个事件的链接会自动取消勾选

### 统一事件操作
1. 点击左侧面板 "事件" 标题旁边的 "统一事件操作" 按钮
2. 在弹出的向量操作弹窗中选择操作类型和参数
3. 点击 "执行" 按钮，操作会应用到所有事件
4. 点击 "取消" 或关闭弹窗则不执行操作

## 7. 预览功能

### 时间节点预览
1. 在右侧预览区域的输入框中输入时间（毫秒或拍，根据当前模式）
2. 点击 "获取时间节点" 按钮
3. 预览画布会显示该时间点的位置坐标
4. 输入框值变化时，滑块会自动同步到对应位置

### 播放预览
1. 点击 "播放" 按钮开始预览
2. 点击 "停止" 按钮停止预览
3. 拖动滑块可以手动调整预览时间

### 分组预览选择
1. 在左侧面板底部的 "分组预览选择器" 中选择要预览的分组
2. 支持连续选择：选择分组1和分组3时，分组2也会自动被选中
3. 点击 "清空选择" 按钮可以取消分组过滤，显示所有事件

## 8. 事件分组功能

### 创建分组
1. 在事件列表中选中要分组的连续事件
   - **重要**：选中事件时必须连续，例如选择事件3和5会自动包含事件4
   - 选中头尾事件后，中间所有事件都会被包含在分组中
2. 点击 "创建分组" 按钮
3. 输入分组名称并确认创建

### 分组操作
- **展开/折叠**：点击分组名称左侧的按钮可以展开或折叠分组
- **重命名**：点击分组名称可以直接编辑
- **解除分组**：点击分组右侧的 "解除分组" 按钮，可以将事件从分组中移除，但保留事件本身
- **删除整个分组**：点击分组右侧的 "删除" 按钮，可以删除整个分组及其所有事件

### 解除全部分组
- 点击 "解除全部分组" 按钮可以一次性移除所有分组
- 所有事件会保留，但分组信息会被清除

### 分组显示规则
- 分组内的所有事件默认一起显示/隐藏
- 展开分组时显示所有事件，折叠时只显示分组摘要
- 分组信息仅保存在本地，不会在JSON输出中体现

## 9. 退出与保存

### 退出应用
1. 点击右上角的 "退出" 按钮
2. 会弹出确认对话框，询问是否保存当前配置：
   - **是**：保存配置到本地，下次启动时自动加载
   - **否**：不保存配置，下次启动时为空
   - **取消**：取消关闭操作，返回应用

### 清空已保存
1. 点击左上角的 "清空已保存" 按钮
2. 会清除所有本地保存的配置
3. 应用会重置为初始状态

## 10. 常见问题

### 问题：BPM模式下时间计算不准确
**解决方案**：确保BPM值设置正确，系统会根据BPM值自动计算实际时间

### 问题：向量操作后链接被取消
**解决方案**：这是正常现象，因为向量操作可能会改变事件的时间和位置，导致链接关系不再适用

### 问题：预览区域坐标显示不正确
**解决方案**：检查事件的起点和终点设置，确保缓动函数选择正确

### 问题：创建分组时事件没有连续
**解决方案**：分组功能要求事件必须连续选择。系统会自动将选中事件的头尾之间的所有事件包含进来。

### 问题：退出后配置没有保存
**解决方案**：只有在退出确认对话框中选择"是"才会保存配置。选择"否"或"取消"不会保存。

## 11. 注意事项

- 请定期保存JSON文本，以防数据丢失
- 大型项目可能会影响性能，建议适当拆分编辑
- 如遇到应用崩溃，请重新启动应用，上次保存的配置会自动加载
- 分组信息仅保存在本地，不会随JSON导出
- 事件连续性是创建分组的前提条件
